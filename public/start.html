Quick Start

Last updated: May 3rd, 2017

Getting Started

First step is to create an account here and create your first application , so you will get your APP Key & APP Secret.

Note about keys

Your App Secret should be strictly confidential, so don't share it with anyone or post it publicly in any way.

Installation

Laravel
Symfony
Codeigniter
Generic PHP

Laravel Framework

$ composer require shieldfy/shieldfy-laravel-client

Add Shieldfy to your service provider list in config/app


'providers' => [
    ...
    Shieldfy\ShieldfyLaravelClient\ShieldfyServiceProvider::class,
],

Publish shieldfy configurations

$ php artisan vendor:publish

don't forget to add your api key, api secret in your .env file


SHIELDFY_APP_KEY=YOUR_APP_KEY
SHIELDFY_APP_SECRET=YOUR_APP_SECRET

Finally add shieldfy global middleware in app/Http/Kernel.php


protected $middleware = [
    ...
    \Shieldfy\ShieldfyLaravelClient\Middlewares\ShieldfyMiddleware::class
];

Generic PHP

Via Composer (Recommended)

Composer is a highly recommended way to manage your application dependecies learn more

$ composer require shieldfy/shieldfy-php-client

Use Shieldfy\Guard in the start of your application


<?php
use Shieldfy\Guard;
$guard = Guard::init([
    'app_key'=>'APP_KEY',
    'app_secret'=>'APP_SECRET',
])->catchCallbacks();

If you are using PDO , attach the object to Shieldfy guard


<?php
$pdo = $guard->attachDB($pdo);

Configurations

Debug level

You can turn debug on/off depend on your needs in the application , the default value is false ( we recommend to leave it false in the production environment )

If debug is false, this will prevent your application from throwing errors related to security or bad server responses for example.

<?php
use Shieldfy\Guard;
Guard::init([
        'app_key'=>'APP_KEY',
        'app_secret'=>'APP_SECRET',
        'debug'=>true //default is false
])->catchCallbacks();

Threat Response (Action)

Shieldfy lets you control the action the package should take when spotting dangerous activities.

Available Actions: ( Block | Listen ) The default action is block which will block the dangerous activities. The other action listen will record the threat and notify you, but will not block the request and will let the user continue to your application.

<?php
use Shieldfy\Guard;
Guard::init([
        'app_key'=>'APP_KEY',
        'app_secret'=>'APP_SECRET',
        'action'=>'listen' //default is block
])->catchCallbacks();

Overide / Disable Headers

For security reasons, Shieldfy exposes number of headers to enforce some security practices.

The headers exposed by default:

  • X-XSS-Protection
  • X-Content-Type-Options
  • X-Frame-Options

If for some reason you don't want these headers to be exposed, or if you want to expose other value for them, you can control it. Just pass an array for the headers and pass the changed values, or false if you want to disable it completely. 

<?php
use Shieldfy\Guard;
Guard::init([
        'app_key'=>'APP_KEY',
        'app_secret'=>'APP_SECRET',
        'headers'=>[
            'X-Frame-Options' => 'DENY', //this header value will be DENY
            'X-XSS-Protection' => false //this header will not be shown
        ]
])->catchCallbacks();
