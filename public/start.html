Quick Start

Last updated: Apr 9, 2018

Getting Started

First step is to create an account here and create your first application , so you will get your APP Key & APP Secret.

Note about keys

Your App Secret should be strictly confidential, so don't share it with anyone or post it publicly in any way.

Installation

Laravel
Codeigniter
Generic PHP

Laravel Framework

$ composer require shieldfy/shieldfy-php-client

If you are using laravel before 5.5 you must add our service provider in config/app


'providers' => [
    ...
    \Shieldfy\Extensions\Laravel\ShieldfyServiceProvider::class,
],

Publish shieldfy configurations

$ php artisan vendor:publish

don't forget to add your api key, api secret in your .env file


SHIELDFY_APP_KEY=YOUR_APP_KEY
SHIELDFY_APP_SECRET=YOUR_APP_SECRET

Codeigniter Framework

$ composer require shieldfy/shieldfy-php-client

Enable hooks in the config application/config/config.php

$config['enable_hooks'] = TRUE;

Add the following lines at the end of application/config/hooks.php file

if(!class_exists(\Composer\Autoload\ClassLoader::class)) require_once(FCPATH.'/vendor/autoload.php');

                                    $shieldfy = \Shieldfy\Guard::init([
                                    'app_key'       => '{{ $thisApp['key'] }}',
                                    'app_secret'    => '{{ $thisApp['secret'] }}'
                                    ]);

                                    $hook['post_controller_constructor'] = \Shieldfy\Extensions\CodeIgniter\Bridge::hook($shieldfy,isset($hook) ? $hook : null);

Generic PHP

$ composer require shieldfy/shieldfy-php-client

Add this line to your script front usually index.php

<?php
if(!class_exists(\Composer\Autoload\ClassLoader::class)) require_once(__DIR__.'/vendor/autoload.php');

$shieldfy = \Shieldfy\Guard::init([
'app_key'       => '{{ $thisApp['key'] }}',
'app_secret'    => '{{ $thisApp['secret'] }}'
]);

 

Attach your Database Object PDO to shieldfy for analysis


//$pdo is the variable that holds your PDO connection
$shieldfy->attachPDO($pdo);

Configurations

Debug level

You can turn debug on/off depend on your needs in the application , the default value is false ( we recommend to leave it false in the production environment )

If debug is false , this will prevent your application to throw errors related to security or bad server response for example.

<?php
\Shieldfy\Guard::init([
        'app_key'=>'APP_KEY',
        'app_secret'=>'APP_SECRET',
        'debug'=>true //default is false
]);

Threat Response (Action)

Shieldfy lets you control the action the package should take when spotting dangerous activities.

Available Actions: ( Block | Listen ) The default action is block which will block the dangerous activities. The other action listen will record the threat and notify you, but will not block the request and will let the user continue to your application.

<?php
\Shieldfy\Guard::init([
        'app_key'=>'APP_KEY',
        'app_secret'=>'APP_SECRET',
        'action'=>'listen' //default is block
]);

Overide / Disable Headers

For security reasons Shieldfy expose number of headers to enforce some security practices.

The headers exposed by default

  • X-XSS-Protection
  • X-Content-Type-Options
  • X-Frame-Options

If for some reason you don't want this headers to be exposed or you want to expose another value for it, you can control it. Just pass array for the headers and pass the changed value or false if you want to disable it completely. 

<?php
\Shieldfy\Guard::init([
        'app_key'=>'APP_KEY',
        'app_secret'=>'APP_SECRET',
        'headers'=>[
            'X-Frame-Options' => 'DENY', //this header value will be DENY
            'X-XSS-Protection' => false //this header will not be shown
        ]
]);
